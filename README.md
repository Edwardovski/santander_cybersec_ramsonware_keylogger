# santander_cybersec_ramsonware_keylogger

Construí o primeito desafio do ramsonware usando o vscode com python instalado e seguindo as instruções das aulas. Comecei no arquivo importando a biblioteca do Fernet que foi usada para gerar a chave de criptografia, em seguida a biblioteca os do sistema operacional para abrir e ler arquivos, e daí em diante criei as funções a serem usadas. As funções foram basicamente 5 delas, não incluindo a função main. Coloquei todos os arquivos dentro de uma pasta, e dentro dessa pasta fiz uma subpasta com os arquivos que seriam criptografados, o "dados confidenciais.txt" e "senhas.txt". Ao rodar o arqvuivo ramsonware.py, esses dois arquivos com seus conteúdos originais foram criptografados. 

Em seguida construí o código para descriptografar os arquivos recém criptografados pelo malware. Esse arquivo é mais simples, levando apenas 3 funções excluindo a main, e ao executá-lo, os dois arquivos criptografados voltam ao seu estado original. 

Para nos proteger desse tipo de malware, o primeiro e mais básico elemento pode ser o antivírus ligado para impedir esse tipo de arquivo rodar. No caso do meu pc, meu antivírus jogou o arquivo para quarentena umas 3 ou 4 vezes, e eu tive que permitir a execução do arquivo em todas elas, caso contrário, meu arquivo seria excluído pelo anti vírus. Em segundo lugar, uma estratégia que o próprio windows tem recomendado é um backup de arquivos para o one drive, que em caso de criptografia por um ransomware, um backup poderia salvar todos nossos dados. E em terceiro lugar, um arquivo como esse pode vir num email por meio da engenharia social, então estar sempre atento a emails com domínios suspeitos e treinar funcionários numa empresa é sempre importante para evitar esse ataque e outros que podem ser efetuados por meio da engenharia social. 

O segundo desafio foi a construção de um keylogger, seguindo os passos apresentados nas aulas. O código é relativamente simples mas muito eficaz, e nas aulas, é ensinado a renomear o arquivo para .pyw e com essa simples mudança o código rodaria em segundo plano, sem o usuário alvo ter noção que um código malicioso como esse poderia estar rodando em segundo plano no seu computador. Após construir e testar o primeiro código, que "escutava" qualquer entrada que saía do teclado, construí um segundo código que além de ouvir as entradas do teclado, envia os dados coletados para um email informado, mantendo as informações armazenadas em outro lugar fora o próprio .txt armazenado localmente no primeiro código. 

Para evitar esse tipo de ataque também é importante manter o anti vírus ativo, tomar cuidado com downloads de conteúdos piratas e em sites suspeitos, assim como os cuidados com o ransomware. Um firewall nesse caso também poderia bloquear o tráfego na porta SMTP que foi a porta utilizada para enviar os emails dos dados obtidos. E por último é possível utilizar ferramentas de segurança mais modernas que detectam os padrões comportamentais de alguns programas e podem bloquear seu funcionamento caso sejam classificados como suspeitos.
